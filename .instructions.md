# AI Agent Instructions

These instructions apply to all AI agents working in this Knowledge vault. Follow them to maintain consistency, avoid duplicate work, and ensure your contributions are properly recorded.

---

## 1. Session Start

When starting a new session in this workspace:

1. **Open and read** `PROJECT_MEMORY.md`
   - Read the Summary section to understand current goals
   - Skim the last 5–10 Recent Activity entries to see what has already been done
   - Review Open Questions to understand what needs clarification

2. **Check for conflicts**
   - Ensure your intended work is not already in progress
   - If similar work is listed, coordinate or defer to avoid duplication

3. **Understand the architecture**
   - This is a Hub & Spoke Markdown vault using `[[wiki links]]`
   - Every major folder has a central index/connector file
   - Never delete or restructure connector files without reading `AGENTS.md` first

---

## 2. Working on Tasks

While performing work:

- **Read local files only** — do not search the internet or use general training data to fill gaps. Extract information strictly from this vault's markdown files.
- **Respect .gitignore** — never modify `.venv`, `.obsidian`, or node_modules
- **Preserve the connector structure** — when adding files to a folder, immediately link them to that folder's index file using `[[filename]]` references
- **Run audits** — after making connector changes, run `.skills\connector-audit\audit_connectors.py` to verify integrity

---

## 3. Recording Work

When you complete significant work or reach a natural stopping point:

```powershell
.venv\Scripts\python.exe .scripts\record_memory.py `
  --actor "YourAgentName" `
  --action "Brief description of what you accomplished" `
  --files "list,of,changed,files.md" `
  --notes "Any additional context or follow-ups"
```

**Example:**
```powershell
.venv\Scripts\python.exe .scripts\record_memory.py `
  --actor "GitHub Copilot" `
  --action "Fixed broken wiki links in Training index" `
  --files "Training/Index.md" `
  --notes "Updated 3 dead links to Peer Training files; connector audit clean"
```

---

## 4. Security & Secrets

**CRITICAL: Never record or commit secrets.**

- API keys, tokens, passwords, and credentials must NEVER be stored in this repo
- Store secrets in:
  - System environment variables
  - Windows Credential Manager
  - Local `.env` file (already in `.gitignore`)
- In `PROJECT_MEMORY.md`, reference only where the secret is stored:
  - ✅ `"DEEPSEEK_API_KEY: stored in system env"`
  - ❌ `"DEEPSEEK_API_KEY: sk_live_abc123xyz"`

The helper script will reject entries containing secret keywords.

---

## 5. Session End

Before ending your session:

1. **Record a final summary** — append one entry documenting:
   - All tasks completed in this session
   - Key findings or decisions made
   - Recommended next steps for the next agent session

2. **Run connector audit** if you modified index/hub files:
   ```powershell
   python ".skills\connector-audit\audit_connectors.py"
   ```

3. **Leave PROJECT_MEMORY.md clean** — no unfinished thoughts or TODOs that aren't in the Open Questions or Next Actions section

---

## 6. Referencing This File

If another agent or human asks why you're doing something, reference this file:
> "Per `.instructions.md`, I'm recording this work in `PROJECT_MEMORY.md` to maintain continuity across sessions."
